Lesson 1: OSS and Open Standards

What is Open Source Software (OSS)?

▼

Software whose source code is publicly available for anyone to study, modify, and distribute

"OSS is when the source code of the software is publicly accessible and anyone can study, modify, and distribute it for any purpose."

Key Characteristics

▼
  • Requires an open source license (Apache 2.0, GPLv3, MIT, etc.)
  • Can be used commercially without licensing fees
  • Encourages community collaboration
  • Promotes transparency and innovation

Major OSS Projects

▼
Linux Kubernetes Terraform Prometheus Django PostgreSQL Node.js React

Open Container Initiative (OCI)

▼

Founded in 2015 to standardize container formats and runtimes.

Analogy

Standardized wheel sizes allow any wheel to fit any car; OCI standards allow any compliant container runtime or image to work together.

OCI Specifications

📦
Image Specification

How container images are built and packaged

⚙️
Runtime Specification

How containers execute and behave

🔄
Distribution Specification

How images are transferred via registries

Why Open Standards Matter

  • ✅ Avoid vendor lock-in
  • ✅ Ensure compatibility across tools
  • ✅ Enable ecosystem growth
  • ✅ Promote interoperability

Lesson 3: CNCF Project Maturity & Governance

Project Maturity Levels

▼

CNCF categorizes projects into three maturity levels:

🧪

Sandbox

  • Early-stage projects
  • Experimental or emerging technologies
  • Limited production use
→
🌱

Incubating

  • Used in production
  • Active contributions
  • Documentation + versioning
→
🎓

Graduated

  • Contributors from multiple orgs
  • Passed security audit
  • Core Infrastructure Initiative best practices
  • Defined governance + committer processes
All CNCF Projects Must
  • Follow the CNCF Code of Conduct
  • Follow the CNCF IP Policy (usually Apache 2.0 + CC BY 4.0)

CNCF Governance Structure

▼

Minimal Viable Governance

Projects self-govern unless intervention is needed. CNCF provides structure without micromanagement.

CNCF Governing Bodies

🏛️ Governing Board (GB)
  • Business decisions
  • Budget management
  • Marketing strategy
🔧 Technical Oversight Committee (TOC)
  • Technical vision
  • Project approval
  • Maturity level decisions
👥 End User Community (EUC)
  • Feedback from real users
  • Use case sharing
  • Community input

🎉 First Graduated Project

Kubernetes was the first project to graduate in 2018.

Lesson 4: Cloud Roles & Personas

Cloud Architect

▼
🏗️

Cloud Architect

The infrastructure designer

  • Designs cloud infrastructure
  • Chooses cloud providers & services
  • Understands CAPEX vs OPEX
  • Ensures resiliency, scalability, automation

DevOps Engineer

▼
🔄

DevOps Engineer

Bridges development + operations

  • Automates CI/CD pipelines
  • Uses tools like Terraform, Ansible, Puppet
  • Often works with Kubernetes
  • Emphasizes shared responsibility & learning over blame
Terraform Ansible Puppet Jenkins GitLab CI

DevSecOps Engineer

▼
🔐

DevSecOps Engineer

DevOps + Security focus

  • Integrates security early in development
  • Shift-left security approach
  • Automates security testing

DevOps = Development + Operations

DevSecOps = Development + Security + Operations

More Cloud Roles

▼
💰
FinOps Engineer

Manages cloud costs, pricing models, budget transparency

🛡️
Site Reliability Engineer (SRE)

Ensures SLAs/SLOs, maintains reliability, on-call rotations

☁️
Cloud Engineer

Deep knowledge of cloud provider services, more cloud-specific than DevOps

📊
Data Engineer

Manages data storage, protection, retention, analytics

💻
Full Stack Developer

Works on frontend + backend, may have basic cloud/DevOps knowledge

Lesson 5: Kubernetes Certification Path

CNCF Kubernetes Certifications

▼

CNCF offers four Kubernetes certifications:

Entry

KCNA

Kubernetes & Cloud Native Associate

  • Multiple-choice
  • Entry-level
  • 3-year validity
→
Developer

CKAD

Certified Kubernetes Application Developer

  • Hands-on
  • Focus on deploying apps
  • 2-year validity
→
Admin

CKA

Certified Kubernetes Administrator

  • Hands-on
  • Focus on cluster management
  • 2-year validity
→
Security

CKS

Certified Kubernetes Security Specialist

  • Hands-on
  • Requires valid CKA
  • Most difficult
  • 2-year validity

KCNA Exam Details

▼
60 Questions
90 Minutes
75% Passing Score
1 Free Retake

Exam Requirements

▼
  • Online proctored exam
  • Requires webcam + microphone
  • Auto-scored within ~24 hours
  • One free retake included

KCNA Exam Domains

▼
Kubernetes Fundamentals 46%
Container Orchestration 22%
Cloud-Native Architecture 16%
Cloud-Native Observability 8%
Cloud-Native App Delivery 8%
